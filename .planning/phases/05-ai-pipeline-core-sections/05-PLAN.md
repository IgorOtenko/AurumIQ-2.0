# Phase 5 — AI Pipeline & Core Sections

Add the Claude-powered analysis layer to AurumIQ. Each AI dashboard section is generated by the same pipeline: fetch the latest finance data, send it to Claude with a section-specific prompt, validate the response (schema + traceability), and persist it for fast re-reads.

## Wave 5A — Foundation (this work)

Shared infrastructure used by every AI section.

- `@anthropic-ai/sdk` installed; `ANTHROPIC_API_KEY` + `ANTHROPIC_MODEL` validated in `src/lib/env.ts`.
- Prisma `Analysis` model with `(userId, ticker, sectionType)` unique key + `sources` snapshot for hallucination auditing.
- `src/lib/ai/` — typed contracts (`types.ts`), Zod validators (`schemas.ts`), Anthropic singleton (`client.ts`), per-section prompt builders (`prompts/*.ts`) sharing a cached `SYSTEM_PROMPT`, traceability check (`validation.ts`), main pipeline (`generate.ts`), browser fetch helpers (`api-client.ts`), Tanstack Query hook (`hooks.ts`).
- `POST /api/ai/[ticker]` streams `progress` → `complete`/`error` SSE events while it fetches data and runs the pipeline. `GET` returns the latest persisted row.

Design decisions:
- Default model `claude-sonnet-4-6` with env override; adaptive thinking + `effort: high` because financial reasoning benefits from both.
- System prompt is stable across all 3 sections to maximize prompt-cache reuse (Sonnet 4.6 prefix cache ≥ 2048 tokens).
- Traceability validator is permissive (substring) — favors catching invented numbers over false positives.
- On `validation_failed`, prior `content` is preserved so the UI never surfaces fresh hallucinations.

## Wave 5B — Three section components (parallel)

Three independent agents implement the dashboard surfaces:
- Bull vs Bear (4+4 bullets + synthesizing one-liner).
- Catalysts & Risks (3–5 bullets each).
- Live on the Call (5–8 listen-for items).

Each agent consumes `generateAnalysisStream()` + `useLatestAnalysis()` and renders progress, refresh, and stale states. No changes to Wave 5A code.
