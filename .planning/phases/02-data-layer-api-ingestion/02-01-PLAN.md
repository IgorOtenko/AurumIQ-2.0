<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API route | Untrusted ticker string and dataType param from user |
| API route -> Yahoo Finance | Outbound request with user-controlled ticker value |
| Yahoo Finance -> Zod validation | Untrusted external API response with volatile shape |
| Cache read -> consumer | JSONB data loses type safety at PostgreSQL boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | API route ticker param | mitigate | Zod tickerSchema validates /^[A-Z0-9.]{1,10}$/ before any processing |
| T-02-02 | Tampering | Yahoo Finance response | mitigate | Zod safeParse on every response; warn-and-store on partial; reject structurally invalid |
| T-02-03 | Tampering | Cache poisoning | mitigate | Zod validation BEFORE setCache; only validated or partially-validated data enters cache |
| T-02-04 | Information Disclosure | API error messages | mitigate | financeLogger.error server-side with full context; NextResponse returns generic "An unexpected error occurred" to client |
| T-02-05 | Spoofing | Unauthenticated API access | mitigate | getServerSession check at route entry; 401 if no session |
| T-02-06 | Denial of Service | Bulk ticker requests | accept | Low risk for demo/solo use; Phase 7 adds rate limiting middleware |
| T-02-SC | Tampering | pnpm install (yahoo-finance2, pino, pino-pretty) | mitigate | All packages verified in Package Legitimacy Audit (02-RESEARCH.md); no [ASSUMED] or [SUS] packages |
</threat_model>
