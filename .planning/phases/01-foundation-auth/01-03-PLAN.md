    Create src/app/api/auth/reset-password/route.ts (POST):
    - Accept JSON body {email: string}
    - Validate with Zod: email is z.string().email()
    - Look up user by email. If user not found, STILL return 200 with same message (prevent email enumeration)
    - If user found: generate a cryptographically random token using Node.js crypto.randomUUID()
    - Delete any existing unused tokens for this email (db.passwordResetToken.deleteMany where email and usedAt is null)
    - Create new PasswordResetToken with expiresAt = now + 1 hour
    - Call sendPasswordResetEmail(email, token)
    - Return 200 JSON {message: "If an account with that email exists, a reset link has been sent"}

    Create src/app/api/auth/reset-password/confirm/route.ts (POST):
    - Accept JSON body {token: string, password: string}
    - Validate with Zod: token is z.string().uuid(), password is z.string().min(8)
    - Look up token via db.passwordResetToken.findUnique({where: {token}})
    - Reject if: token not found, token.expiresAt < now, token.usedAt is not null
    - On valid token: hash new password with bcryptjs.hash(password, 12)
    - Update user: db.user.update({where: {email: token.email}, data: {hashedPassword}})
    - Mark token used: db.passwordResetToken.update({where: {id: token.id}, data: {usedAt: new Date()}})
    - Return 200 JSON {message: "Password reset successfully"}
    - On invalid/expired token: return 400 JSON {error: "Invalid or expired reset token"}
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/auth/reset-password | Email address from untrusted client |
| browser -> /api/auth/reset-password/confirm | Token and new password from untrusted client |
| URL query param -> confirm page | Token in URL could be intercepted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Information Disclosure | /api/auth/reset-password | mitigate | Always return 200 with same message regardless of email existence (prevents email enumeration) |
| T-01-08 | Spoofing | reset token | mitigate | Token is crypto.randomUUID() (128-bit entropy); expires in 1 hour; single-use (usedAt marked) |
| T-01-09 | Tampering | /api/auth/reset-password/confirm | mitigate | Zod validates token as UUID format and password min 8; bcrypt hash with cost 12 |
| T-01-10 | Replay | reset token reuse | mitigate | usedAt field checked before accepting; token marked used atomically with password update |
| T-01-11 | Information Disclosure | reset link in console | accept | Dev-only behavior; production will use SES (Phase 6). Console logging is appropriate for development |
| T-01-SC | Tampering | npm installs | mitigate | No new packages installed in this plan |
</threat_model>
